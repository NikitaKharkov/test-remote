CustomizingÂ Publication Finder in EBSCOadmin

The new Publication Finder is customizable by making changes to the code found in the Bottom Branding settings in EBSCOadmin.

Note: HTML code will not work in the Bottom Branding field for Publication Finder profiles.

To customize Publication Finder:

  1. Log into EBSCOadmin at: https://eadmin.ebscohost.com.

  2. Select the Publication Finder profile you would like to modify from the Choose Profile drop-down menu.

  3. Click the Branding sub-tab.

  4. Under Bottom Branding, click the Modify link.

  5. Click the Add New Branding link.

  6. Enter a Branding Version Name.

  7. Select the appropriate Type of Text radio button to indicate the format for the custom text (either HTML or Plain Text.)

  8. In Enter Custom Text field, note the customization code that has been entered for your profile.

    See the Customization table below to learn where to make your changes.

Customization Settings and Explanations

The values you can customize to configure Publication Finder are highlighted in red.

Branding Configuration Code Details
"Locale": "en", Set locale to configure the Publication Finder interface for various regions/languages. For example, setting locale to "ar" configures the interface for Arabic.
"PfSearchOptions": {
"ShowComponent": true,
"SearchFieldDefault": "titlename",
"ShowSearchFieldControl": true,
"SearchTypeDefault": "contains",
"ShowSearchTypeControl": true,
"SubjectSchema": "tli"
}

This section configures the various search parameters.

  • If "ShowComponent" is set to true, then the search box is shown on Publication Finder. If "ShowComponent" is set to false, then the search box is not shown.

  • SearchFieldDefault: This value can be used to set to the default selection for the search field drop-down menu. Enter one of the following values: "titlename" "subject" "isxn" "publisher"

  • ShowSearchFieldControl: If this is set to true, the search field drop-down menu will be shown on Publication Finder. If it is set to false, the search field drop-down menu will not be shown on the interface.

  • SearchTypeDefault: This value can be used to set the default selection for the search type drop-down menu. Enter one of the following values: "Any" "Contains" "Proxmity" "Exact Phrase" "Exact Match"

  • ShowSearchTypeControl: If this is set to true, the search type drop-down menu is shown on Publication Finder. If it is set to false, the search type drop-down menu is not shown on the interface.

  • SubjectSchema: This is to set the subject scheme. Valid Values are tli/medical/lc.
"PfResults": {
"SortByCoverage": true
}

This is to configure how Full Texts results are sorted.

  • If SortByCoverage is set to true, then Full Texts links for a given record set will be sorted by coverage date.

  • If SortByCoverage is set to false, then Full Texts links for a given record set will be sorted by the Package Name.
"PfStyleOptions": {
"PrimaryColor": "#228B22"
}

This value is used to set the background color of the Publication Finder user interface. Note: This value does not customize the color of the result list view.

The Google Color Picker Tool can help you find a color code to use in this field.
"PfFacets": {
"PfAlpha": {
"ShowComponent": true,
"ShowSubMenu": true,
"AlphaMenuLanguage": "en"
},
"PfResource": {
"ShowComponent": true
},
"PfSubject": {
"ShowComponent": true,
"EnableSearch": true
},
"PfDatabase": {
"ShowComponent": true,
"EnableSearch": true
},
"PfPeerReview": {
"ShowComponent": true
}
}

This is to configure what facets to show and hide. The Publication Finder user interface supports following facets:

  • PfAlpha = Alphamenu
    • If showsubmenu is set to true, the submenu will be shown. If it is set to false, the submenu will not be shown.
    • AlphaMenuLanguage is used to set the language for the alphamenu. For example, the value "ja" would display the Japanese language alphamenu.
  • PfResource = Resource Types facet
    • If ShowComponent is set to true, the Resource Type facet will be shown under Filters in the result list view. If ShowComponent is set to false, the Resource Type facet will not be shown in the result list view.
    • If EnableSearch is set to true, the user can search for facet values inside the Resource Type facet. For example, if EnableSearch is set to false, the search capability for this facet will not be shown.
  • PfSubject = Subject facet
    • If ShowComponent is set to true, the Subject facet will be shown in the result list view. If ShowComponent is set to false, the subject facet will not be shown in the result list view.
    • If EnableSearch is set to true, the user can search for facet values inside the Subject facet. For example, if EnableSearch is set to false, the search capability for this facet will not be shown.
  • PfDatabase = Database facet
    • If ShowComponent is set to true, the Database facet will be shown in the result list view. If ShowComponent is set to false, the Database facet will not be shown in the result list view.
    • If EnableSearch is set to true, the user can search for facet values inside the database facet. For example, if EnableSearch is set to false, the search capability for this facet will not be shown.
  • PfPeerReview = Peer reviewed facet
    • If ShowComponent is set to true, the Peer Reviewed facet will be shown in the result list view. If ShowComponent is set to false, the Peer Reviewed facet will not be shown in the result list view.
"PfBrowseOptions": {
"ShowDatabases": true,
"ShowSubjects": true
}

This section of code enables you to show or hide the Database and Subject browse components of Publication Finder.

  • If ShowDatabase is set to true, the "Browse Database" feature is shown on Publication Finder. If ShowDatabase is set to false, the "Browse Database" feature is NOT displayed.

  • If ShowSubjects is set to true, the "Browse Subject" feature is shown on Publication Finder. If it is set to false, the "Browse Subject" feature is NOT displayed.
