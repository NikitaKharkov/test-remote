Holdings Upload Matching - Internal Information


What is the Resolve Ambiguous Matches Feature?
When a holdings file is uploaded without identifiers (KBIDs), the system automatically matches uploaded titles to items in the EBSCO knowledge base using Title Name, Package, URL etc.  Sometimes more than one potential match is identified and the system doesn’t know which one to select.  With the Resolve Ambiguous Matches feature, you can review the potential matches and manually select the correct one.

Ambiguous matches can also result from the Match Custom Titles to EBSCO's Global Knowledge Base feature.  You can use Resolve Ambiguous Matches to select the best match for these titles, as well.


When Can I Resolve Ambiguous Matches?
When the most recent Holdings Upload or Custom Matching contains at least one ambiguously matched title, the Resolve Ambiguous Matches link will appear on the Overview.  Click the link to access the ambiguously matched titles and select your matches.  You can work through your items until you are finished or until another Holdings Upload or Custom Matching is processed with at least one ambiguously matched title.  If this happens, your matching session will be “closed” meaning no additional matching on that load is possible.  But don’t worry, all the matching you have done prior to the load being closed has been applied.


How is Matching Applied?
As soon as you select a match for your title, the match is applied.  There is no ability to undo your match at this time.


Are My Match Selections Remembered?
The system doesn’t support matching memory at this time.  This means that if you upload a file without IDs, match the ambiguously matched items and then re-upload a similar file (again without IDs), you will need to re-select the appropriate matches.


If I Complete All the Matching, is the Entire Upload Matched?
Maybe not.  The Resolve Ambiguous Matches feature currently only allows the selection of matches for items where the system has identified multiple potential matches.  There may be other items in your upload for which the system was unable to find any match at all.  Those items cannot be matched using this feature at this time.


How Does this Feature Related to the Error File?
This feature allows you to manually select matches for all items in your error file that have an error of “Duplicate Title Match”. 


Who Can Use this Feature?
In order to use this feature, you must have rights to the Holdings Matching role in Gold Admin.  So this is an EBSCO internal only feature at this time.  We plan to open this feature up to customers after the second release.

Matching algorithms can’t match every title, however.  Some titles are unmatched because we can’t match the Package or because we don’t have a similar title in our knowledge base.  Other titles can’t be automatically matched because we find multiple potential matches.
